# Security Policy

## Supported Versions

Only the latest release on the `main` branch receives security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | ✅                |
| older   | ❌                |

## Reporting a Vulnerability

If you discover a security vulnerability in feed-manager-ui, please report it
privately **before** filing a public issue.

**Email**: sweetbeex@babesky.com

Please include:

- A brief description of the issue
- Steps to reproduce
- The potential impact
- Any suggested remediation (optional)

You should receive an acknowledgment within 48 hours. We'll keep you informed
of the progress toward a fix.

### What to expect

1. **Acknowledgment** — we confirm receipt within 2 business days
2. **Investigation** — we assess severity and impact
3. **Fix** — we develop and test a patch
4. **Release** — we ship a fix and publish an advisory

We'll credit you in the advisory if you'd like (otherwise we'll keep your
report anonymous).

## Scope

This policy covers the feed-manager-ui web application (this repo). For
vulnerabilities in the feed generator platform itself, see
[Issues in feedgen](https://github.com/Babesky-com/feedgen/security/policy).
